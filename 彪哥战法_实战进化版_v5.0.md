### 模块1：周期判断系统（从定性到定量）

#### 旧版（v1.0-v2.0）：定性描述
- 春播：强势股补跌≥10%，跌停>30家
- 夏长：涨停>100家，连板≥4板

#### v5.0：定量评分系统（0-100分）+ 监管风险预警（独立）

```python
class CycleScoringSystem:
    """周期量化评分系统"""
    
    def calculate_cycle_score(self, market_data):
        """
        计算当前市场周期得分（满分100分）
        注：监管风险不入评分，作为独立预警系统
        """
        scores = {
            # 春播信号（权重30%）
            '跌停家数': min(market_data['跌停'] / 50 * 30, 30),
            '强势股补跌': min(market_data['强势股补跌幅度'] / 15 * 20, 20),
            '连板高度': max(0, (5 - market_data['连板高度']) / 5 * 15),
            '恐慌指数': market_data['恐慌度'] * 0.35,
            
            # 夏长信号（权重40%）
            '涨停家数': min(market_data['涨停'] / 100 * 40, 40),
            '量能水平': min(market_data['量能'] / 30000 * 20, 20),
            '板块共振': market_data['主线板块数'] * 5,
            
            # 秋收信号（权重20%）
            '炸板率': min(market_data['炸板率'] / 35 * 20, 20),
            '高潮分化': market_data['后排掉队率'] * 10,
            
            # 冬藏信号（权重10%）
            '流动性': max(0, (30000 - market_data['量能']) / 30000 * 10),
        }
        
        total_score = sum(scores.values())
        return total_score, scores
    
    def determine_season(self, score):
        """根据得分判断季节"""
        if score < 20:
            return "❄️ 冬藏期", "极低仓位（<10%），保命为主"
        elif score < 40:
            return "🌱 春播期", "轻仓试错（<30%），寻找新主线"
        elif score < 70:
            return "🍂 秋收期", "减仓兑现（<40%），去弱留强"
        else:
            return "☀️ 夏长期", "重仓持有（<70%），让利润奔跑"


class RegulatoryRiskSystem:
    """监管风险预警系统（独立不入评分）"""
    
    def check_regulatory_risk(self, market_data):
        """
        监管是结果，不是原因
        监控导致监管的前置因子
        """
        red_flags = 0  # 红灯条件
        yellow_flags = 0  # 黄灯条件
        
        # 🔴 红灯条件（监管即将出手）
        if market_data['连板高度'] > 10:
            red_flags += 1
        if market_data['20cm连板'] > 5:
            red_flags += 1
        if market_data['10日涨幅'] > 150:
            red_flags += 1
        if market_data['批量100%异动'] > 3:
            red_flags += 1
        if market_data['天地板反包']:
            red_flags += 1
            
        # 🟡 黄灯条件（监管可能关注）
        if 7 <= market_data['连板高度'] <= 10:
            yellow_flags += 1
        if 3 <= market_data['20cm连板'] <= 4:
            yellow_flags += 1
        if 100 <= market_data['7日涨幅'] <= 150:
            yellow_flags += 1
        if market_data['板块成交占比'] > 25:
            yellow_flags += 1
        if market_data['特停个案']:
            yellow_flags += 1
        
        # 判定预警级别
        if red_flags >= 3:
            return "🔴 红灯", "监管即将出手，仓位上限20%", 0.20
        elif yellow_flags >= 2 or red_flags >= 1:
            return "🟡 黄灯", "监管可能关注，仓位上限50%", 0.50
        else:
            return "🟢 绿灯", "监管无动作，正常仓位", 1.00
```

### 模块3：龙头识别系统（四维锁定法）

#### 龙头识别四要素（权重分配）

| 要素 | 权重 | 核心指标 | 一票否决项 |
|:---|:---:|:---|:---|
| **题材正宗性** | 40% | 业务占比、技术壁垒、订单实锤 | 蹭概念、PPT公司 |
| **盘子适中** | 30% | 流通市值30-200亿最佳 | 太大（>500亿）拉不动、太小（<20亿）易被控 |
| **股性活跃** | 20% | 历史涨停次数、连板基因、辨识度 | 常年横盘、无涨停记忆 |
| **技术位置** | 10% | 突破形态、量价配合 | 高位放量滞涨 |

#### 龙头演变规律（基于商业航天案例）

```
题材发酵期（首板-3板）：
├── 多家竞争，不确定性高
├── 观察标的：率先涨停+封单坚决+带动跟风
└── 策略：小仓位试错多只，去弱留强

题材确认期（4板-7板）：
├── 龙头浮出水面，跟风分化
├── 核心特征：卡位成功、主动性强、带动板块
└── 策略：聚焦龙头，断板反包加仓

题材高潮期（8板+）：
├── 龙头独舞，跟风乏力
├── 风险信号：监管关注、炸板率上升、后排掉队
└── 策略：3板后减仓，5板清仓
```

### 模块4：中军识别系统（三维度筛选）

**中军定义**：板块内市值较大（>200亿）、成交活跃（>10亿/日）、走势稳健的核心标的。

#### 中军识别三维度

| 维度 | 标准 | 作用 |
|:---|:---|:---|
| **正宗性** | 主营业务与题材高度相关，非边缘业务 | 防「伪中军」陷阱 |
| **承载力** | 日均成交>10亿，可容纳大资金进出 | 大资金战场 |
| **稳定性** | 波动率<板块平均，沿5日线上行 | 板块定海神针 |

#### 中军与龙头关系

```
龙头（情绪载体）        中军（资金载体）
    ↑                      ↑
  连板高度              趋势新高
  带动情绪              稳定军心
    ↓                      ↓
  短期爆发力            中期持续性
  游资主导              机构+游资
```

**实战要点**：
- 龙头断板时，中军不破位 = 板块未结束
- 中军跌破10日线 = 板块退潮信号
- 龙头+中军共振 = 最佳做多窗口

### 模块5：买卖点决策系统（多周期共振）

#### 战略层（月线）：定四季大方向
- 月线MACD金叉 + 成交量放大 = 战略做多
- 月线MACD死叉 + 成交量萎缩 = 战略防守

#### 战术层（周线）：定板块主线
- 周线突破 + 板块指数新高 = 战术进攻
- 周线震荡 + 板块分化 = 战术观望

#### 执行层（日线）：定具体买卖点

```python
class TradingSignalSystem:
    """交易信号生成系统"""
    
    def generate_buy_signals(self, stock_data, market_cycle):
        """生成买入信号"""
        signals = []
        
        # 春播期买点
        if market_cycle == '春播':
            if self.check_spring_buy_point(stock_data):
                signals.append({
                    'type': '春播第一买',
                    'price': stock_data['突破价位'],
                    'position': 0.1,  # 10%仓位
                    'reason': '底部放量突破颈线'
                })
            elif self.check_spring_second_buy(stock_data):
                signals.append({
                    'type': '春播第二买',
                    'price': stock_data['回踩价位'],
                    'position': 0.2,  # 加仓至30%
                    'reason': '回踩颈线企稳'
                })
        
        # 夏长期买点
        elif market_cycle == '夏长':
            if self.check_summer_confirm_buy(stock_data):
                signals.append({
                    'type': '夏长确认买',
                    'price': stock_data['主升浪启动价'],
                    'position': 0.4,  # 满仓70%
                    'reason': '主升浪启动+板块共振'
                })
        
        return signals
    
    def generate_sell_signals(self, stock_data, market_cycle):
        """生成卖出信号"""
        signals = []
        
        # 秋收期卖点
        if market_cycle == '秋收':
            if stock_data['高位放量滞涨']:
                signals.append({
                    'type': '秋收第一卖',
                    'position': -0.3,  # 减仓至40%
                    'reason': '高位放量滞涨'
                })
            elif stock_data['双顶跌破颈线']:
                signals.append({
                    'type': '秋收第二卖',
                    'position': -0.2,  # 清仓
                    'reason': '双顶跌破颈线'
                })
        
        return signals
```

### 模块6：风控与仓位管理系统

#### 仓位管理矩阵（结合监管风险）

**基础仓位（四季标准）**：

| 季节 | 总仓位 | 单票仓位 | 加减仓策略 |
|:---|:---:|:---:|:---|
| **冬藏** | <10% | <3% | 空仓观望，或极轻仓试盘 |
| **春播** | <30% | <10% | 首仓10%，确认后加仓至30% |
| **夏长** | <70% | <20% | 主升浪确认后满仓，不做T |
| **秋收** | <40% | <15% | 逐步减仓，去弱留强 |

**监管风险调整（动态上限）**：

| 监管预警 | 仓位上限调整 | 策略调整 |
|:---:|:---:|:---|
| **🟢 绿灯** | 按四季标准执行 | 正常操作 |
| **🟡 黄灯** | 上限50%（无论季节） | 去弱留强，只做龙头 |
| **🔴 红灯** | 上限20%（无论季节） | 清仓或极轻仓避险 |

**实际仓位计算公式**：
```
实际仓位 = min(四季标准仓位, 监管风险上限仓位)

例1：夏长期（70%）+ 🔴红灯（20%）= 实际仓位20%
例2：春播期（30%）+ 🟡黄灯（50%）= 实际仓位30%（不受影响）
例3：夏长期（70%）+ 🟢绿灯（100%）= 实际仓位70%
```

## 💡 五、v5.0 核心心法

1. **顺势而为，四季轮回**
   > 不是预测市场，而是识别市场所处阶段，然后顺势而为

2. **多维度验证，不单一信号**
   > 周期+题材+龙头+中军+买卖点，多维度共振才出手

3. **看图形识周期**
   > 春播W底、夏长阶梯涨、秋收双顶、冬藏急跌，图形会说话

4. **多周期共振，定买卖**
   > 战略层定方向、战术层定板块、执行层定买卖点

5. **风控第一，盈利第二**
   > 冬藏不抄底、春播不追高、夏长不做T、秋收不贪心

6. **监管是结果，不是原因**
   > 连板高度、涨幅、批量异动才是前置因子，提前预警比被动应对更重要

7. **量化时代，适应进化**
   > 从散户情绪周期到量化资金周期，从4小时到4分钟，战法必须持续进化
