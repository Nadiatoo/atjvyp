### 模块1：周期判断系统（从定性到定量）

#### 旧版（v1.0-v2.0）：定性描述
- 春播：强势股补跌≥10%，跌停>30家
- 夏长：涨停>100家，连板≥4板

#### v5.0：定量评分系统（0-100分）

```python
class CycleScoringSystem:
    """周期量化评分系统"""
    
    def calculate_cycle_score(self, market_data):
        """
        计算当前市场周期得分
        满分100分，不同季节对应不同分数区间
        """
        scores = {
            # 春播信号（权重30%）
            '跌停家数': min(market_data['跌停'] / 50 * 30, 30),
            '强势股补跌': min(market_data['强势股补跌幅度'] / 15 * 20, 20),
            '连板高度': max(0, (5 - market_data['连板高度']) / 5 * 15),
            '恐慌指数': market_data['恐慌度'] * 0.35,
            
            # 夏长信号（权重40%）
            '涨停家数': min(market_data['涨停'] / 100 * 40, 40),
            '量能水平': min(market_data['量能'] / 30000 * 20, 20),
            '板块共振': market_data['主线板块数'] * 5,
            
            # 秋收信号（权重20%）
            '炸板率': min(market_data['炸板率'] / 35 * 20, 20),
            '监管信号': market_data['监管强度'] * 10,
            
            # 冬藏信号（权重10%）
            '流动性': max(0, (30000 - market_data['量能']) / 30000 * 10),
        }
        
        total_score = sum(scores.values())
        return total_score, scores
    
    def determine_season(self, score):
        """根据得分判断季节"""
        if score < 20:
            return "❄️ 冬藏期", "极低仓位（<10%），保命为主"
        elif score < 40:
            return "🌱 春播期", "轻仓试错（<30%），寻找新主线"
        elif score < 70:
            return "🍂 秋收期", "减仓兑现（<40%），去弱留强"
        else:
            return "☀️ 夏长期", "重仓持有（<70%），让利润奔跑"
```

### 模块2：题材挖掘系统（六维评估框架）

基于商业航天深度分析，建立标准化题材评估体系：

```
┌─────────────────────────────────────────────────────────────┐
│                    超级题材六维评估模型                       │
├─────────────────────────────────────────────────────────────┤
│ 维度        权重    评估要点                    评分标准      │
├─────────────────────────────────────────────────────────────┤
│ 时代背景    20%    国家战略/国际竞争/产业方向   0-20分       │
│ 产业阶段    20%    技术突破/商业化前夜/产业链   0-20分       │
│ 政策强度    15%    顶层设计/资金规模/政策周期   0-15分       │
│ 催化事件    15%    可跟踪里程碑/持续性/验证性   0-15分       │
│ 市场环境    15%    大盘位置/流动性/竞争格局     0-15分       │
│ 资金认可    15%    机构覆盖/游资介入/成交放大   0-15分       │
├─────────────────────────────────────────────────────────────┤
│ 总分：0-100分                                                │
│ 90-100分 = 超级题材（如商业航天）                             │
│ 70-89分  = 大题材（如AI、机器人）                             │
│ 50-69分  = 中等题材（轮动级别）                               │
│ <50分    = 小题材（一日游）                                   │
└─────────────────────────────────────────────────────────────┘
```

**应用示例：商业航天评分 93分（超级题材）**
- 时代背景 20/20：国家战略级
- 产业阶段 19/20：可回收技术突破临界点
- 政策强度 14/15：央地联动，万亿级市场
- 催化事件 13/15：可回收火箭、订单落地
- 市场环境 13/15：牛市氛围，流动性充足
- 资金认可 14/15：机构+游资共振

### 模块3：龙头识别系统（四维锁定法）

#### 龙头识别四要素（权重分配）

| 要素 | 权重 | 核心指标 | 一票否决项 |
|:---|:---:|:---|:---|
| **题材正宗性** | 40% | 业务占比、技术壁垒、订单实锤 | 蹭概念、PPT公司 |
| **盘子适中** | 30% | 流通市值30-200亿最佳 | 太大（>500亿）拉不动、太小（<20亿）易被控 |
| **股性活跃** | 20% | 历史涨停次数、连板基因、辨识度 | 常年横盘、无涨停记忆 |
| **技术位置** | 10% | 突破形态、量价配合 | 高位放量滞涨 |

#### 龙头演变规律（基于商业航天案例）

```
题材发酵期（首板-3板）：
├── 多家竞争，不确定性高
├── 观察标的：率先涨停+封单坚决+带动跟风
└── 策略：小仓位试错多只，去弱留强

题材确认期（4板-7板）：
├── 龙头浮出水面，跟风分化
├── 核心特征：卡位成功、主动性强、带动板块
└── 策略：聚焦龙头，断板反包加仓

题材高潮期（8板+）：
├── 龙头独舞，跟风乏力
├── 风险信号：监管关注、炸板率上升、后排掉队
└── 策略：3板后减仓，5板清仓
```

### 模块4：中军识别系统（三维度筛选）

**中军定义**：板块内市值较大（>200亿）、成交活跃（>10亿/日）、走势稳健的核心标的。

#### 中军识别三维度

| 维度 | 标准 | 作用 |
|:---|:---|:---|
| **正宗性** | 主营业务与题材高度相关，非边缘业务 | 防「伪中军」陷阱 |
| **承载力** | 日均成交>10亿，可容纳大资金进出 | 大资金战场 |
| **稳定性** | 波动率<板块平均，沿5日线上行 | 板块定海神针 |

#### 中军与龙头关系

```
龙头（情绪载体）        中军（资金载体）
    ↑                      ↑
  连板高度              趋势新高
  带动情绪              稳定军心
    ↓                      ↓
  短期爆发力            中期持续性
  游资主导              机构+游资
```

**实战要点**：
- 龙头断板时，中军不破位 = 板块未结束
- 中军跌破10日线 = 板块退潮信号
- 龙头+中军共振 = 最佳做多窗口

### 模块5：买卖点决策系统（多周期共振）

#### 战略层（月线）：定四季大方向
- 月线MACD金叉 + 成交量放大 = 战略做多
- 月线MACD死叉 + 成交量萎缩 = 战略防守

#### 战术层（周线）：定板块主线
- 周线突破 + 板块指数新高 = 战术进攻
- 周线震荡 + 板块分化 = 战术观望

#### 执行层（日线）：定具体买卖点

```python
class TradingSignalSystem:
    """交易信号生成系统"""
    
    def generate_buy_signals(self, stock_data, market_cycle):
        """生成买入信号"""
        signals = []
        
        # 春播期买点
        if market_cycle == '春播':
            if self.check_spring_buy_point(stock_data):
                signals.append({
                    'type': '春播第一买',
                    'price': stock_data['突破价位'],
                    'position': 0.1,  # 10%仓位
                    'reason': '底部放量突破颈线'
                })
            elif self.check_spring_second_buy(stock_data):
                signals.append({
                    'type': '春播第二买',
                    'price': stock_data['回踩价位'],
                    'position': 0.2,  # 加仓至30%
                    'reason': '回踩颈线企稳'
                })
        
        # 夏长期买点
        elif market_cycle == '夏长':
            if self.check_summer_confirm_buy(stock_data):
                signals.append({
                    'type': '夏长确认买',
                    'price': stock_data['主升浪启动价'],
                    'position': 0.4,  # 满仓70%
                    'reason': '主升浪启动+板块共振'
                })
        
        return signals
    
    def generate_sell_signals(self, stock_data, market_cycle):
        """生成卖出信号"""
        signals = []
        
        # 秋收期卖点
        if market_cycle == '秋收':
            if stock_data['高位放量滞涨']:
                signals.append({
                    'type': '秋收第一卖',
                    'position': -0.3,  # 减仓至40%
                    'reason': '高位放量滞涨'
                })
            elif stock_data['双顶跌破颈线']:
                signals.append({
                    'type': '秋收第二卖',
                    'position': -0.2,  # 清仓
                    'reason': '双顶跌破颈线'
                })
        
        return signals
```

### 模块6：风控与仓位管理系统

#### 仓位管理矩阵

| 季节 | 总仓位 | 单票仓位 | 加减仓策略 |
|:---|:---:|:---:|:---|
| **冬藏** | <10% | <3% | 空仓观望，或极轻仓试盘 |

| **秋收** | <40% | <15% | 逐步减仓，去弱留强 |

## 💡 五、v5.0 核心心法

1. **顺势而为，四季轮回**
   > 不是预测市场，而是识别市场所处阶段，然后顺势而为

2. **多维度验证，不单一信号**
   > 周期+题材+龙头+中军+买卖点，多维度共振才出手

3. **看图形识周期**
   > 春播W底、夏长阶梯涨、秋收双顶、冬藏急跌，图形会说话

4. **多周期共振，定买卖**
   > 战略层定方向、战术层定板块、执行层定买卖点

5. **风控第一，盈利第二**
   > 冬藏不抄底、春播不追高、夏长不做T、秋收不贪心

6. **量化时代，适应进化**
   > 从散户情绪周期到量化资金周期，从4小时到4分钟，战法必须持续进化
