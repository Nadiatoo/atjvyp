## 一、周期权重动态调整优化

### 1.1 建立市场环境识别模型

**量化指标定义**：

```python
# 量化主导指标
量化席位占比 > 40% 且 日内ATR > 5% 且 涨停次日低开率 > 60%

# 机构主导指标  
北向连续5日净流入 且 机构调研密度 > 均值2倍

# 游资主导指标
连板高度 > 8板 且 换手率 > 20%
```

**历史回测找出最优权重**：

| 权重组合 | 量化主导胜率 | 机构主导胜率 | 游资主导胜率 |
|:---:|:---:|:---:|:---:|
| 基准(30/40/20/10) | 62% | 71% | 58% |
| 量化A(35/35/20/10) | **68%** | 69% | 60% |
| 机构A(25/50/15/10) | 61% | **75%** | 55% |

### 1.2 动态权重切换规则

```python
切换条件：
1. 市场环境判定变化
2. 且距离上次切换 >= 10天（冷却期）
3. 新环境持续3天确认（防抖动）

切换后：记录切换时间、原因、预期效果
```

## 四、中军动态调整优化

### 4.1 切换阈值优化

**原方案**：强度分差15%切换
**问题**：可能太敏感

**优化方案**：
```python
切换条件：
1. 新中军强度分 > 旧中军 × 1.15（强15%）
2. 且新中军连续3天领先（防单日异常）
3. 且距离上次切换 >= 5天（冷却期）

不切换场景：
- 旧中军虽落后但仍在板块前5
- 且板块整体处于主升浪（不折腾）
```

### 4.2 分级中军制度

```
主中军（70%仓位）：最强标的，趋势稳健
副中军（30%仓位）：次强标的，弹性更大

切换策略：
- 主中军落后 → 副中军晋升为主
- 而不是直接换掉主中军
```

## 五、买卖点精确量化优化

### 5.1 ATR自适应止损

**替代固定百分比**：
```python
ATR = 真实波动幅度（14日）
止损位 = 买入价 - 2×ATR

好处：
- 波动大的股票（如科创板）止损宽松
- 波动小的股票（如银行股）止损紧凑
```

### 5.2 成交量确认机制

```python
突破买点确认：
- 突破当日成交量 > 5日均量 × 1.5
- 且阳线实体 > 3%

回踩买点确认：
- 回踩当日成交量 < 5日均量 × 0.7
- 且收小阳线或十字星
```

### 5.3 多时间框架验证

```
日线：判断大方向（春播/夏长/秋收）
↓
小时线：找具体买卖点
↓
15分钟：确认入场时机

示例：
日线突破 + 小时线金叉 + 15分钟放量 = 高置信度买点
```

## 六、系统性改进机制

### 6.1 置信度评分系统

所有信号输出置信度（0-100%）：

| 置信度 | 仓位调整 | 说明 |
|:---:|:---:|:---|
| >80% | 100%仓位 | 高置信度，全力干 |
| 60-80% | 70%仓位 | 中等置信度，留余地 |
| 40-60% | 40%仓位 | 低置信度，小仓位试错 |
| <40% | 不操作 | 放弃，等更好机会 |

**置信度计算**：
```python
置信度 = 信号强度 × 市场环境匹配度 × 历史胜率

例：
- 周期判断置信度 = 85% → 正常仓位
- 龙头识别置信度 = 55% → 仓位减半
```

### 6.2 失效案例库建立
